# Love Forge Taste System

## Purpose
Turn subjective quality into objective publish tests so weak pages never ship.

## Core principle
Do not ask readers to invent. Give them a next step they can use tonight.

## 80/20 product focus (current)
Prioritize guided action loops over content expansion:
1) Get My 7-Day Plan flow
2) Immediate scheduling + partner invite
3) Simple score tracking (before/after)
4) Early safety/escalation routing
5) Next rep defined at every endpoint

## The 6 Gates (all required)

### Gate 1: Tonight Test
A tired parent can start in 60 seconds.
- Clear first action above the fold
- Copy/paste script included
- Concrete starter artifact (plan, checklist, template, or builder)
- Fallback mode for low energy / kid chaos

### Gate 2: Emotional Resonance Test
The page feels human and hopeful.
- Opens with a relatable real-life problem
- Reduces shame (“normal and fixable” language)
- Sounds like a trusted friend, not a lecture
- Gives agency in first screen

### Gate 3: Practicality Test
The advice is executable.
- Time-bound steps (5/10/15 minute options)
- One issue at a time (no overload)
- Measurable outcome for 7 days
- “What to do if awkward” guidance

### Gate 4: Clarity Test
No confusion points.
- No jargon without explanation
- No vague verbs (“improve connection”) without exact behavior
- Every “create/build/plan” has a ready-to-use template/builder

### Gate 5: Visual Test
Page is skimmable.
- No long text walls
- Visual block every 2-3 text blocks
- Mobile-first readability
- CTA clarity (one primary next step)

### Gate 6: Proof-of-Ship Test
No “done” without evidence.
- Commit hash
- Pages status built
- Live URL posted

## Scoring
- Score each gate 0/1.
- Minimum ship threshold: 6/6.
- If 5/6 or below: revise before shipping.

## Decision rule
Usefulness > cleverness. Clarity > novelty. Actionability > completeness.
