# Love Forge Design Standards

## UX principles
- 5-second clarity: users instantly know where to start
- One-screen action: every page has a visible next action above the fold
- Visual hierarchy: headings, cards, and buttons should guide scanning
- Reduce wall-of-text: break dense sections with tiles, checklists, and callouts

## Accessibility
- Minimum contrast AA
- Touch targets >= 44px
- Clear focus states
- Avoid color-only meaning

## Layout checklist (publish gate)
- [ ] Hero has one promise + one clear CTA
- [ ] At least one visual block every 2-3 text blocks
- [ ] Mobile view tested
- [ ] Reading level plain English
- [ ] Script and metric included
- [ ] Every tool page includes either (a) an exact plan or (b) a quick action checklist

## Deployment gate
No design change counts as done without:
1. Commit hash
2. GitHub Pages status built
3. Live URL posted


## Anti-text-wall rule (required)
- Blog hub cannot be a plain list only.
- Every high-traffic page must include at least one visual block (tiles/cards/icons) above the fold.
- Break any block over 5 lines into bullets or cards.
- Keep instruction manuals skimmable: max 4 bullets, one sentence each.
- Prefer progressive disclosure: short default content first, details second.

## Actionability standard (required)
- Do not force minute-by-minute agendas on tools that don't need them.
- If a tool is conversational/structured, include an exact plan.
- If a tool is habit/setup-based, include a quick action checklist.
- Reader should know exactly what to do in under 60 seconds.
