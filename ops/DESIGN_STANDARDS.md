# Love Forge Design Standards

## UX principles
- 5-second clarity: users instantly know where to start
- One-screen action: every page has a visible next action above the fold
- Visual hierarchy: headings, cards, and buttons should guide scanning
- Reduce wall-of-text: break dense sections with tiles, checklists, and callouts

## Accessibility
- Minimum contrast AA
- Touch targets >= 44px
- Clear focus states
- Avoid color-only meaning

## Layout checklist (publish gate)
- [ ] Hero has one promise + one clear CTA
- [ ] At least one visual block every 2-3 text blocks
- [ ] Mobile view tested
- [ ] Reading level plain English
- [ ] Script and metric included

## Deployment gate
No design change counts as done without:
1. Commit hash
2. GitHub Pages status built
3. Live URL posted
