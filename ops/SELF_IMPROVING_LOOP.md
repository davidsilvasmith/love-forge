# Self-Improving Loop (Love Forge)

## North Star
"People forward this to someone they care about."

## Primary Metric
- **Share Intent Rate** = % of visitors who click a "Send to spouse/friend" action.

## Secondary Metrics
- Time on page > 2:30
- Scroll depth > 70%
- Return within 7 days
- Email/lead opt-in

## Loop (Weekly)

1. **Capture Signals**
   - Top exits, bounce pages, low-scroll pages
   - User comments/messages about confusion or usefulness

2. **Diagnose Problem**
   - Label each issue: Clarity, Relevance, Actionability, Trust, UX friction

3. **Design Small Experiments**
   - 1 homepage experiment
   - 2 blog template experiments
   - 1 CTA experiment

4. **Ship Fast**
   - Publish within 24-48 hours

5. **Ship + Verify (mandatory)**
   - Commit + push to `main`
   - Confirm GitHub Pages status = `built`
   - Post deploy proof: commit hash + changed files + live URL

6. **Measure**
   - Compare metrics after 3-7 days

7. **Decide**
   - Keep / Kill / Iterate

## Content Quality Gate (must pass all)
- Is the problem concrete?
- Is there a 10-minute action?
- Is there a script users can copy?
- Is there a measurable outcome?
- Does this feel helpful before it feels salesy?

## Rule
No new feature unless it improves one metric tied to usefulness or trust.
